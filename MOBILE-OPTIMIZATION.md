# Pixel-Master 移动端优化说明

## 优化内容总览

### 1. 响应式布局
- **桌面端（>768px）**: 保持原有的左右分栏布局
- **移动端（≤768px）**: 改为上下布局，画布在上，控制面板在下
- **小屏手机（≤480px）**: 进一步紧凑布局，优化空间利用
- **横屏模式**: 自动调整画布和控制面板的比例

### 2. 触摸事件支持

#### 单指拖动
- 在画布上单指滑动即可平移图像
- 流畅的拖动手势，无延迟

#### 双指缩放
- 双指捏合/展开手势进行缩放
- 以双指中心点为缩放中心
- 缩放范围：20% - 500%
- 实时显示缩放百分比

#### 颜色选择模式
- 启用颜色选择模式后，点击画布上的像素可选择颜色
- 触摸事件不会与平移/缩放冲突

### 3. UI 优化

#### 触摸区域优化
- 所有按钮最小高度 44px（符合 iOS 人机界面指南）
- 颜色样本增大到 28px，便于精确点击
- 滑块控件高度增加，易于操作

#### 字体优化
- 输入框字体大小 16px，防止 iOS 自动缩放
- 标题和文字大小适配小屏幕

#### 布局优化
- 尺寸输入框在小屏幕上垂直排列
- 抖动算法选择器支持横向滚动
- 替换操作按钮全宽显示

### 4. 下载按钮修复
- 移动端固定在底部，距离边缘 1em
- 左右边距各 1em，方便单手操作
- 确保在上传图片后正确显示

### 5. 其他改进
- 添加 viewport meta 标签，正确控制页面缩放
- 防止 iOS Safari 的弹性滚动干扰
- 禁用悬停效果，改用激活状态反馈
- 语言选择器位置优化，不遮挡主要内容

## 使用指南

### 移动端操作

#### 平移图像
1. 单指在画布上滑动即可平移

#### 缩放图像
1. 双指捏合缩小
2. 双指展开放大
3. 或使用底部的缩放滑块

#### 选择颜色
1. 勾选"启用颜色选择模式"
2. 点击画布上的像素点
3. 选择要替换的颜色

#### 下载结果
1. 处理完图像后，底部的"下载当前效果图"按钮会亮起
2. 点击即可下载 PNG 格式的处理结果

## 技术细节

### CSS 媒体查询
```css
/* 平板和手机 */
@media screen and (max-width: 768px) { ... }

/* 小屏手机 */
@media screen and (max-width: 480px) { ... }

/* 横屏模式 */
@media screen and (max-width: 768px) and (orientation: landscape) { ... }

/* 触摸设备 */
@media (hover: none) and (pointer: coarse) { ... }
```

### JavaScript 触摸事件
- `touchstart`: 检测触摸开始
- `touchmove`: 处理拖动和缩放手势
- `touchend`: 结束触摸状态
- `passive: false`: 允许阻止默认行为

### Viewport 设置
```html
<meta name="viewport" content="width=device-width, initial-scale=1.0, maximum-scale=5.0, user-scalable=yes, viewport-fit=cover">
```
